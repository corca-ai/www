# Blog integration

The `/blog` section is a static copy of the standalone Corca Blog project at
`/Users/koleuka/Documents/corca-blog-pages`. Keep that project as the source of
truth for blog content UI, client-side behavior, post publishing scripts and
post metadata. This repo vendors the generated output under `public/blog/`, then
uses the main website header and footer shell so `/blog` stays visually aligned
with the rest of `www.corca.ai`.

## What is copied

The vendored output includes:

- `public/blog/index.html` — the blog home content shell with the main website
  header and footer applied.
- `public/blog/app.js` and `public/blog/styles.css` — the original client app
  and visual system, including search, sorting, saved posts, recent reads,
  pagination, reading settings, reading progress, reactions, share/download
  actions and analytics hooks.
- `public/blog/posts/` — generated static post pages plus `posts/index.json`.
- `public/blog/assets/` — the image and icon assets referenced by the copied
  app and post pages.
- `public/blog/rss.xml`, `public/blog/feed.json`, `public/blog/sitemap.xml` and
  `public/blog/robots.txt` — the feed and discovery files generated by the blog
  project.

## Cloudflare asset contract

Blog data is deployed as Cloudflare Workers Static Assets, not as Worker source,
D1, KV or a runtime filesystem. The website Worker uses the `ASSETS` binding to
serve and read blog files from the built `dist/blog/` directory:

- The public blog app reads `/blog/posts/index.json` directly from static
  assets.
- Static post pages are served from `/blog/posts/<slug>/index.html`.
- The admin read APIs use `env.ASSETS.fetch()` to read
  `/blog/posts/index.json` and `/blog/admin/post-sources/<slug>.html`.
- Direct browser access to `/blog/admin/post-sources/*.html` is blocked by the
  Worker; those files are present only so the authenticated admin API can read
  source content from Cloudflare assets.

## Refresh from the source project

Build the source project with `/blog` as the base path, then copy its `dist/`
directory into this repo:

```bash
cd /Users/koleuka/Documents/corca-blog-pages
CORCA_SITE_URL=https://www.corca.ai/blog node scripts/build-cloudflare-pages.js
cd /Users/koleuka/Documents/www
cp -R /Users/koleuka/Documents/corca-blog-pages/dist/. public/blog
```

For sandboxed agent runs, copy the source project to a writable temporary
directory first and run the same build command there. The important invariant is
that generated links and asset URLs start with `/blog/`, not `/`.

## Routing contract

Astro does not define `/blog` pages in `src/pages/`. Cloudflare Workers Static
Assets serves the files from `public/blog/` directly:

- `/blog` and `/blog/` load the copied blog home.
- `/blog/posts/<slug>/` loads the copied static post page.
- `/blog/posts/index.json` powers the copied client app.
- `/blog/rss.xml` and `/blog/feed.json` are copied from the source build.

The main site redirects in `public/_redirects` keep legacy blog URLs pointing at
the `/blog/posts/<slug>/` paths.
