# Internationalization

The site ships four locales: Korean at the root (`/`), English under `/en/`,
Japanese under `/ja/` and Simplified Chinese under `/zh/`. It uses Astro's
built-in i18n routing with no prefix on the default Korean locale, matching the
original site's URLs.

## Where content lives

- `src/i18n/ui.ts` — the locale list plus navigation, footer and home strings.
- `src/i18n/pageMeta.ts` — per-page SEO titles and descriptions for the static
  pages; product pages carry their own per-locale copy in their manifest ([products](products.md)).
- `src/i18n/structuredData.ts` — localized schema.org nodes (breadcrumbs and the
  product `SoftwareApplication`) merged into each page's JSON-LD.
- `src/content/` — news and colleague entries as YAML, one file each, translated
  per locale and validated by a schema.
- `src/i18n/utils.ts` — helpers such as `localizePath` and locale detection.

Each page is authored once as a component and rendered per locale by a single
dynamic route, as described in [architecture](architecture.md). Adding an article, colleague or
locale is walked through in the [runbook](runbook.md). Translation edits flow into the
build like any other change, so run the checks in [development](development.md) afterwards.
